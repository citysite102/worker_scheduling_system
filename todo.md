**問題流程**：
1. 前端使用 `setHours()` 設定本地時區的時間（09:00 台灣時間）
2. JavaScript 自動轉換為 UTC 時間（01:00 UTC）
3. 傳送到後端後，MySQL 再次轉換，變成 2026-02-19 17:00:00 UTC
4. 結果：demands 表中的時段是 09:00-10:00，assignments 表中的時段是 01:00-02:00
5. 可行性判斷時，比較 09:00-10:00 與 01:00-02:00，發現不重疊
6. 所以宇軒被標記為「可指派」，但實際上他已經被指派到需求單 390142

**影響範圍**：
- 所有的指派記錄都有這個問題
- 排班衝突檢測失效
- 實際工時回填可能也有時間不一致的問題

**解決方案**：
- [x] 修改前端 DemandDetail.tsx 的時間轉換邏輯，使用 setUTCHours() 而不是 setHours()
- [x] 檢查其他可能有類似問題的地方：
